# 블로그 포스트 데이터베이스 설정 가이드

이 문서는 블로그 시스템을 위한 노션 데이터베이스 설정 방법을 설명합니다.

## 📋 개요

블로그 포스트 데이터베이스는 블로그의 모든 포스트를 관리하고, 발행 상태와 숨김 처리를 제어할 수 있도록 설계되었습니다.

## 🗄️ 데이터베이스 생성

### 1. 새 데이터베이스 생성
1. 노션에서 새 페이지를 생성합니다
2. `/`를 입력하고 "Database"를 선택합니다
3. "Table - Inline" 또는 "Table - Full page"를 선택합니다
4. 데이터베이스 이름을 "Blog Posts" 또는 "블로그 포스트"로 설정합니다

### 2. 데이터베이스 ID 복사
1. 데이터베이스 URL에서 데이터베이스 ID를 복사합니다
2. URL 형식: `https://www.notion.so/your-workspace/DATABASE_ID?v=...`
3. `DATABASE_ID` 부분을 복사하여 환경변수에 설정합니다

## 🏗️ 속성(Properties) 설정

다음 속성들을 순서대로 생성하고 설정해주세요:

### 1. Name (제목)
- **속성명**: `Name`
- **속성 타입**: `Title` (제목)
- **설명**: 블로그 포스트의 제목

### 2. Description (설명)
- **속성명**: `Description`
- **속성 타입**: `Rich Text` (서식 있는 텍스트)
- **설명**: 포스트의 간단한 설명 또는 요약

### 3. Published (발행일)
- **속성명**: `Published`
- **속성 타입**: `Date` (날짜)
- **설명**: 포스트가 발행된 날짜 (발행 여부를 나타냄)
- **설정**: "Include time" 체크

### 4. Tags (태그)
- **속성명**: `Tags`
- **속성 타입**: `Multi-select` (다중 선택)
- **설명**: 포스트에 관련된 태그들
- **옵션 예시**:
  - `React`
  - `Next.js`
  - `TypeScript`
  - `JavaScript`
  - `Web Development`
  - `Tutorial`
  - `Self Introduce`

### 5. Created (생성일)
- **속성명**: `Created`
- **속성 타입**: `Date` (날짜)
- **설명**: 포스트가 처음 생성된 날짜
- **설정**: "Include time" 체크

### 6. Hidden (숨김 처리) - 새로 추가
- **속성명**: `Hidden`
- **속성 타입**: `Checkbox` (체크박스)
- **설명**: 포스트를 숨김 처리할지 여부
- **기본값**: 체크 해제 (false)

### 7. AllowComments (댓글 허용) - 새로 추가
- **속성명**: `AllowComments`
- **속성 타입**: `Checkbox` (체크박스)
- **설명**: 해당 포스트에서 댓글을 허용할지 여부
- **기본값**: 체크됨 (true)

## 🔧 환경 변수 설정

`.env.local` 파일에 다음 환경 변수를 추가하세요:

```env
# 기존 환경 변수들...
NOTION_DATABASE_ID=your_blog_database_id_here
```

### 데이터베이스 ID 찾기:

1. 노션에서 생성한 블로그 포스트 데이터베이스 페이지를 엽니다.
2. URL에서 데이터베이스 ID를 복사합니다.
   - URL 예시: `https://www.notion.so/your-workspace/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=...`
   - 데이터베이스 ID: `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` (32자리 문자열)

## 📝 포스트 작성 방법

### 1. 새 포스트 생성
1. 노션 데이터베이스에서 새 페이지 생성
2. 제목, 설명, 태그 등 메타데이터 입력
3. 본문에 다양한 블록 타입 추가 (텍스트, 이미지, 코드 등)

### 2. 포스트 발행
1. `Published` 속성에 발행 날짜 설정
2. `Hidden` 속성은 체크 해제 상태로 유지
3. 포스트가 블로그에 자동으로 표시됩니다

### 3. 포스트 숨김 처리
1. `Hidden` 속성을 체크하여 숨김 처리
2. 포스트가 블로그에서 사라집니다
3. 관리자 페이지에서는 여전히 관리 가능

### 4. 포스트 초안 저장
1. `Published` 속성을 비워두면 초안 상태
2. `Hidden` 속성과 관계없이 초안으로 처리
3. 블로그에 표시되지 않음

## 🔍 관리자 기능

관리자 페이지(`/admin/posts`)에서 다음 기능을 사용할 수 있습니다:

- **포스트 목록 조회**: 모든 포스트 (발행/초안/숨김) 확인
- **발행 토글**: 포스트 발행/초안 상태 변경
- **숨김 토글**: 포스트 숨김/공개 상태 변경
- **댓글 허용 토글**: 포스트별 댓글 허용/비활성화 설정
- **포스트 삭제**: 포스트 완전 삭제 (아카이브)
- **검색 및 필터**: 제목, 설명으로 검색, 상태별 필터링
- **통계 확인**: 상태별 포스트 개수 확인

## ⚠️ 주의사항

1. **Hidden 속성 추가**: 기존 데이터베이스에 `Hidden` 속성을 추가해야 합니다
2. **AllowComments 속성 추가**: 기존 데이터베이스에 `AllowComments` 속성을 추가해야 합니다
3. **기본값 설정**: 
   - `Hidden` 속성의 기본값을 `false`로 설정하세요
   - `AllowComments` 속성의 기본값을 `true`로 설정하세요
4. **권한 확인**: 노션 API 키가 데이터베이스 수정 권한을 가지고 있는지 확인하세요
5. **백업**: 중요한 포스트는 정기적으로 백업하세요

## 🚀 테스트

설정이 완료되면 다음을 확인합니다:

1. 블로그 메인 페이지에서 발행된 포스트가 표시되는지 확인
2. 숨김 처리된 포스트가 블로그에서 사라지는지 확인
3. 관리자 페이지에서 모든 포스트가 표시되는지 확인
4. 관리자 페이지에서 발행/숨김 토글이 작동하는지 확인
5. 관리자 페이지에서 댓글 허용 토글이 작동하는지 확인
6. 댓글 허용이 비활성화된 포스트에서 댓글 작성이 차단되는지 확인
7. 댓글 허용이 활성화된 포스트에서 댓글 작성이 가능한지 확인
