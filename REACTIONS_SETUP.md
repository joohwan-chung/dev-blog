# 반응 시스템 설정 가이드

이 가이드는 블로그에 반응 시스템(좋아요, 싫어요, 추천, 비추천)을 설정하는 방법을 설명합니다.

## 1. 노션 데이터베이스 생성

### 반응 데이터베이스 생성

1. 노션에서 새 데이터베이스를 생성합니다.
2. 데이터베이스 이름을 "Reactions" 또는 "반응"으로 설정합니다.
3. 다음 속성들을 추가합니다:

#### 필수 속성들:

| 속성 이름 | 속성 타입 | 설명 |
|-----------|-----------|------|
| PostId | Title | 블로그 포스트 ID |
| ReactionType | Select | 반응 타입 (like, dislike, recommend, not_recommend) |
| UserIp | Rich Text | 사용자 IP 주소 |
| CreatedAt | Date | 반응 생성일시 |

#### Select 속성 설정 (ReactionType):

ReactionType 속성의 Select 옵션들을 다음과 같이 설정합니다:

- **like** (좋아요)
- **dislike** (싫어요)  
- **recommend** (추천)
- **not_recommend** (비추천)

## 2. 환경 변수 설정

`.env.local` 파일에 다음 환경 변수를 추가합니다:

```env
# 기존 환경 변수들...
NOTION_REACTIONS_DATABASE_ID=your_notion_reactions_database_id_here
```

### 데이터베이스 ID 찾기:

1. 노션에서 생성한 반응 데이터베이스 페이지를 엽니다.
2. URL에서 데이터베이스 ID를 복사합니다.
   - URL 예시: `https://www.notion.so/your-workspace/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=...`
   - 데이터베이스 ID: `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` (32자리 문자열)

## 3. API 권한 설정

노션 API 키가 다음 권한을 가지고 있는지 확인합니다:

- **Read content**: 데이터베이스 읽기
- **Insert content**: 새 페이지 생성
- **Update content**: 기존 페이지 수정

## 4. 테스트

설정이 완료되면 다음을 확인합니다:

1. 블로그 포스트 페이지에서 반응 버튼이 표시되는지 확인
2. 각 반응 버튼을 클릭했을 때 카운트가 증가하는지 확인
3. 같은 반응을 다시 클릭했을 때 취소되는지 확인
4. 다른 반응을 클릭했을 때 기존 반응이 변경되는지 확인

## 5. 문제 해결

### 일반적인 문제들:

1. **반응 버튼이 표시되지 않음**
   - 환경 변수 `NOTION_REACTIONS_DATABASE_ID`가 올바르게 설정되었는지 확인
   - 노션 API 키 권한 확인

2. **반응 클릭 시 오류 발생**
   - 브라우저 개발자 도구의 콘솔에서 오류 메시지 확인
   - API 엔드포인트 `/api/reactions`가 정상 작동하는지 확인

3. **중복 반응이 생성됨**
   - IP 주소 추출 로직 확인
   - 노션 데이터베이스의 UserIp 속성이 올바르게 설정되었는지 확인

### 로그 확인:

개발 환경에서 다음 명령어로 로그를 확인할 수 있습니다:

```bash
npm run dev
```

콘솔에서 API 호출 관련 로그를 확인하여 문제를 진단할 수 있습니다.

## 6. 데이터 관리

### 반응 데이터 확인:

노션 데이터베이스에서 다음 정보를 확인할 수 있습니다:

- 각 포스트별 반응 개수
- 사용자별 반응 이력
- 반응 생성 시간
- 반응 타입별 분포

### 데이터 정리:

필요시 노션 데이터베이스에서 직접 데이터를 삭제하거나 수정할 수 있습니다.

## 7. 보안 고려사항

- IP 주소는 개인정보이므로 적절히 보호되어야 합니다
- 필요시 IP 주소를 해시화하여 저장하는 것을 고려해보세요
- 노션 데이터베이스 접근 권한을 적절히 관리하세요
